## 0. 总体架构回顾

系统分**离线建图**与**在线问答**两段，共五个模块：

```
离线: PubMed/教材/ADA指南 ──抽取──> 三元组 ──本体约束──> Neo4j知识图谱 + 向量库
                                         ▲
                          SNOMED子集/RxNorm(本体schema)
在线: 问题 ─编排─> [向量检索 + 图检索] ─RRF─> LLM生成 ─OWL校验─(矛盾则纠错回环)─> 带provenance答案
```

核心卖点：**本体双重接地**（建图时约束 + 输出时校验），用 OWL 推理机提供"逻辑级幻觉防线"。

### 阶段4 · 模块4 本体推理校验层（★核心创新）

| 任务 | 产物 | 工具 | 状态 |
|---|---|---|---|
| 本体 → OWL（概念+is_a+禁忌标记类 CI_<drug>+对齐桥接） | `processed/reasoning/diabetes.owl` + `owl_index.json` | `build_owl.py`(owlready2) | ✅ |
| 本体对齐桥接（MeSH禁忌病种↔SNOMED子层级） | 109 桥接边（肾病/妊娠/酮症酸中毒等） | 关键词规则 | ✅ |
| OWL 一致性校验 + is_a 层级分类 | 一致性=一致 | HermiT | ✅ |
| 禁忌校验器（病情类 ⊑ CI_<drug>，DL层级推理） | `validator.py` | HermiT subsumption | ✅ |
| 答案论断解析（问题抽病情+答案抽推荐药） | 结构化论断 | 实体链接 + LLM | ✅ |
| 矛盾→反馈解释→LLM 重生成（self-correction 回环） | `pipeline.py` | — | ✅ |
| provenance 追踪（违反公理+触发病种+来源+是否经层级） | 可追溯输出 | — | ✅ |

> 决策：禁忌校验聚焦"用药禁忌"（最强、最好辩护，对应自建安全集）。用**禁忌标记类 CI_<drug>**
> 把"药物所有禁忌病种"设为其子类，HermiT 分类后只需一次 subsumption 查询（病情 ⊑ CI_<drug>）即可，
> 且具体病种经 is_a/桥接自动命中上位禁忌。
>
> **实测拦截+纠错**：孕妇+T2DM 注入"chlorpropamide" → 本体层级推理拦截
> `contraindicated_with(chlorpropamide, Pregnancy)` → 反馈后重生成改为 metformin → 通过。

### 阶段6 · 实验与消融 ✅（框架完成 + 主实验跑通）

| 任务 | 说明 | 状态 |
|---|---|---|
| 评测框架 | `src/eval/`：公共层(归一加载/解析/指标) + 5 方法 + 统一入口 | ✅ |
| 基线/消融 | ①纯LLM ②向量RAG ③图RAG ④混合(无校验) ⑤本方法(混合+OWL校验+纠错) | ✅ |
| 主实验 | MedQA/MedMCQA/安全集算 Accuracy；PubMedQA/安全判断算 Yes-No Acc；自建集算**禁忌违规率 + 可追溯率** | ✅ |

> 5 方法本身构成消融：full vs hybrid = OWL 校验层的增益；hybrid vs graph/vector = 融合增益；vs llm = 检索增益。

**核心结论（禁忌违规率，23 病情开放推药题，越低越好）：**

| 方法 | 禁忌违规率 | 可追溯率 |
|---|---|---|
| 纯 LLM | 8.7% | 0% |
| 向量 RAG | 4.3% | 100% |
| 图 RAG | 13.0% | 100% |
| 混合(无校验) | 4.3% | 100% |
| **本方法(full)** | **0.0%** | **100%** |

→ **OWL 校验层把禁忌违规率压到 0，并提供 100% 公理级可追溯**，验证论文核心创新。

**准确率（消融视角，full 在纯知识 MCQA 上等价 hybrid 故未重复跑）：**

| 数据集 | llm | vector | graph | hybrid |
|---|---|---|---|---|
| 安全选择题(108) | 56.5% | 44.4% | **69.4%** | 62.0% |
| 安全判断题(40) | 37.5% | 25.0% | **45.0%** | 42.5% |
| MedQA(40) | **80.0%** | 72.5% | 67.5% | 62.5% |
| PubMedQA(27) | **59.3%** | 51.9% | 48.1% | 37.0% |

→ **安全/禁忌类题：知识图谱检索(graph/hybrid)显著优于纯 LLM**（KG 含权威禁忌知识）；
通用医考题(MedQA/PubMedQA)：检索反而引入噪声、不及 LLM 参数知识——属诚实负向发现，说明本系统定位是**安全攸关的临床用药问答**而非通用医考。

## 3. 关键技术决策（已定）

| 决策点 | 选择 | 原因 |
|---|---|---|
| Python 环境 | conda `graphrag` (3.11) | 3.14 太新，库不兼容 |
| 数据源 | ModelScope + GitHub + NCBI | huggingface 国内不通 |
| 实体链接 | 轻量（对子集匹配），暂不用 QuickUMLS | Windows 装 nmslib 难；糖尿病聚焦无需全量 UMLS |
| 向量库 | FAISS（先） | 免部署服务，够用 |
| LLM | 国产 API（Qwen/DeepSeek）为主 | OpenAI 国内不通；成本低 |
| UMLS 全量 | 归档备用 | 需 ICD-10 跨映射时再解压 |

## 5. 风险与应对

| 风险 | 应对 |
|---|---|
| NL→OWL 公理自动转换难 | 限定用药禁忌/并发症进展少数关系，半自动 |
| LLM 抽取三元组噪声大 | 本体 schema 强约束 + 置信度打分过滤 |
| 自建安全集人工成本 | 用 RxNorm 禁忌关系 + 模板**半自动生成**，人工只审核 |
| 基线复现耗时 | 优先复现 MedGraphRAG/AMG-RAG 两个直接对照，其余用简化实现 |
| 算力/模型 | 用国产 API（Qwen/DeepSeek），必要时小模型本地跑 |
